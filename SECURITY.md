# Security

## Reporting a Vulnerability

Please report security issues through the project owner's preferred private channel or GitHub security reporting when available.

Include:

- A clear description of the issue.
- Steps to reproduce.
- Potential impact.
- Relevant environment details.

## Sensitive Data

Do not commit secrets, private keys, API tokens, personal data, runtime databases, logs, or local environment files.

Use example configuration files for documentation and keep real local configuration outside version control.

## Runtime Data

Runtime data, generated files, logs, caches, local databases, and build artifacts should not be committed unless the project owner explicitly defines them as source assets.

# Plan Template

## 1. 需求文档

写用户能理解的非技术说明：真实问题、用户、使用体验、范围、完成标准。

## 2. 当前事实

只写当前仓库、命令输出和 owner 输入确认的事实。

## 3. 失败测试

列出哪些情况说明任务失败。失败测试必须对应真实产品行为或真实工程风险。

## 4. 目标

写最终要交付的可验收结果。

## 5. 不做范围

明确本轮不处理的边界。

## 6. 设计

说明主链路、模块边界、文件职责、状态归属、数据流、错误边界、测试和文档影响。

## 7. 实施任务

- [ ] T001 ...

## 8. 验证计划

写最后怎么证明完成。

```powershell
<完整验证命令>
```

## 9. 收口

记录实际验证命令、结果、已完成事实和剩余风险。
